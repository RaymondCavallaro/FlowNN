# Self, Values, And Curiosity

[Portugues](../pt/concepts/self-values-and-curiosity.md)

This is a later-stage concept note. It should not become a module before the lower layers exist.

The useful ordering is:

```text
continuity of experience
-> self-model
-> self-boundary
-> consequence attribution
-> preference stability
-> value formation
-> value hierarchy
```

Values should not be injected first. A value is not only:

```text
this outcome is good
```

It is closer to:

```text
this outcome matters to the continuity, integrity, or preferred future of this self
```

## Self-Reference Frame

Before values, the system needs a stable self-reference frame:

```text
I was this
I changed because of that
I remain connected across time
```

Without self-continuity, values collapse into external reward labels.

The self-boundary then separates:

```text
me / not-me
my action / external event
my state / other state
my consequence / observed consequence
```

That boundary is what makes consequence attribution possible.

## Consequence Attribution

The system needs to remember not only outcomes, but how outcomes related to actors, choices, and constraints:

```text
event memory
-> actor model
-> available-options model
-> inner simulation
-> outcome comparison
-> emotional valuation
-> value hierarchy update
```

The important addition is option-space. Without it, the system can only learn:

```text
bad result -> bad choice
good result -> good choice
```

With option-awareness, it can distinguish:

```text
bad result, few options -> lower blame / more compassion
bad result, better option existed -> regret
good result, harmful alternatives ignored -> lower praise
good result, better-aligned option chosen -> reinforcement
```

Values are shaped by experienced preference under constrained alternatives, not by outcome labels alone.

A compact event record for later value learning:

```text
Circumstance C
Actor A
Options O1, O2, O3
Chosen option O2
Outcome R
Counterfactual outcomes R1, R3
Self/value impact comparison
Emotional signal
Hierarchy update
```

This lets "regret", "forgiveness", "happiness", and "sorryness" become training signals over simulated alternatives instead of direct labels attached to results.

The order should be:

```text
set coherence
-> constraint detection
-> option-space comparison
-> value-weighted choice
```

## Curiosity

Curiosity should not be a blind always-on drive.

In this architecture it is better framed as:

```text
reduce uncertainty about states that may affect self-continuity or value satisfaction
```

Before self-continuity exists, curiosity is only exploratory behavior. It becomes a value only after the system learns:

```text
exploration -> improved future states
```

Then it can become:

```text
I value discovering
```

## Curiosity As Routing Bias

Curiosity can be expressed with existing pressure-network terms:

```text
if uncertainty is high
and potential impact is high
then temporarily reduce resistance to unexplored paths
```

Exploitation follows established low-resistance paths. Curiosity temporarily allows higher-resistance paths so the system does not converge too early.

This connects directly to landscape probing:

```text
current flow -> known descent path
curiosity -> lateral or uphill probe
```

## Value Conflict

Curiosity must conflict with other values:

```text
curiosity vs safety
curiosity vs stability
curiosity vs efficiency
curiosity vs commitment
```

Too much curiosity prevents consolidation. Too little traps the system in local minima. The hierarchy should eventually decide:

```text
when to explore
when to commit
when to stop
```
