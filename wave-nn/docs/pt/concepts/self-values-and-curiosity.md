# Self, Valores E Curiosidade

[English](../../concepts/self-values-and-curiosity.md)

Esta e uma nota conceitual para uma fase posterior. Ela nao deve virar modulo antes das camadas inferiores existirem.

A ordem util e:

```text
continuidade de experiencia
-> modelo de self
-> fronteira do self
-> atribuicao de consequencia
-> estabilidade de preferencia
-> formacao de valor
-> hierarquia de valores
```

Valores nao devem ser injetados primeiro. Um valor nao e apenas:

```text
este resultado e bom
```

E mais perto de:

```text
este resultado importa para a continuidade, integridade ou futuro preferido deste self
```

## Quadro De Referencia Do Self

Antes de valores, o sistema precisa de um quadro estavel de auto-referencia:

```text
eu era isto
eu mudei por causa daquilo
eu continuo conectado atraves do tempo
```

Sem continuidade do self, valores viram apenas rotulos externos de recompensa.

A fronteira do self separa:

```text
eu / nao-eu
minha acao / evento externo
meu estado / outro estado
minha consequencia / consequencia observada
```

Essa fronteira torna atribuicao de consequencia possivel.

## Atribuicao De Consequencia

O sistema precisa lembrar nao apenas resultados, mas como resultados se relacionam com atores, escolhas e restricoes:

```text
memoria de evento
-> modelo de ator
-> modelo de opcoes disponiveis
-> simulacao interna
-> comparacao de resultados
-> valoracao emocional
-> atualizacao da hierarquia de valores
```

A adicao importante e o espaco de opcoes. Sem ele, o sistema aprende apenas:

```text
resultado ruim -> escolha ruim
resultado bom -> escolha boa
```

Com consciencia de opcoes, ele pode distinguir:

```text
resultado ruim, poucas opcoes -> menos culpa / mais compaixao
resultado ruim, opcao melhor existia -> arrependimento
resultado bom, alternativas danosas ignoradas -> menos elogio
resultado bom, opcao mais alinhada escolhida -> reforco
```

Valores sao moldados por preferencia experienciada sob alternativas restritas, nao so por rotulos de resultado.

Um registro compacto de evento para aprendizagem posterior de valores:

```text
Circumstance C
Actor A
Options O1, O2, O3
Chosen option O2
Outcome R
Counterfactual outcomes R1, R3
Self/value impact comparison
Emotional signal
Hierarchy update
```

Isso permite que "arrependimento", "perdao", "felicidade" e "sorryness" virem sinais de treino sobre alternativas simuladas em vez de rotulos diretos grudados nos resultados.

A ordem deve ser:

```text
coerencia de conjuntos
-> deteccao de restricoes
-> comparacao do espaco de opcoes
-> escolha ponderada por valores
```

## Curiosidade

Curiosidade nao deve ser um impulso cego sempre ligado.

Nesta arquitetura, ela e melhor descrita como:

```text
reduzir incerteza sobre estados que podem afetar continuidade do self ou satisfacao de valores
```

Antes da continuidade do self, curiosidade e apenas comportamento exploratorio. Ela vira valor quando o sistema aprende:

```text
exploracao -> melhores estados futuros
```

Entao pode se tornar:

```text
eu valorizo descobrir
```

## Curiosidade Como Vies De Roteamento

Curiosidade pode ser expressa nos termos da rede de pressao:

```text
se incerteza e alta
e impacto potencial e alto
entao reduzir temporariamente a resistencia de caminhos pouco explorados
```

Exploracao segue caminhos de baixa resistencia ja estabelecidos. Curiosidade permite temporariamente caminhos de maior resistencia para o sistema nao convergir cedo demais.

Isso se conecta diretamente ao landscape probing:

```text
fluxo atual -> caminho de descida conhecido
curiosidade -> probe lateral ou subida local
```

## Conflito De Valores

Curiosidade deve conflitar com outros valores:

```text
curiosidade vs seguranca
curiosidade vs estabilidade
curiosidade vs eficiencia
curiosidade vs compromisso
```

Curiosidade demais impede consolidacao. Curiosidade de menos prende o sistema em minimos locais. A hierarquia deve eventualmente decidir:

```text
quando explorar
quando consolidar
quando parar
```
